# AgePlan — כלי ציות תכנוני לסביבת גיל

## מטרה
אפליקציית ווב לאדריכלים ומתכננים עירוניים.
בדיקת ציות אוטומטית לתדריך תכנון סביבת מגורים לעת זקנה — משרד הבינוי והשיכון, 2024.
המערכת בודקת פרמטרים כמותיים מול ערכי סף מוגדרים ומפיקה דוח ציות לצירוף להיתר בניה.

---

## טכנולוגיות

- React 18 + TypeScript
- Tailwind CSS
- Vite (bundler)
- RTL מלא — כל הממשק בעברית, כיוון rtl
- jsPDF לייצוא דוחות PDF

---

## עקרונות עיצוב (חובה לכל רכיב)

- ללא אייקונים גרפיים — מילים בלבד
- RTL מלא: `dir="rtl"` על כל container ראשי
- פונטים גדולים: body מינימום 15px, כותרות מינימום 18px
- גריד מלא ללא שוליים מיותרים — `w-full`, `h-full`
- שני לוחות צבעים בלבד:
  - ירוק-בינוי: `#1D9E75` (עובר), `#E1F5EE` (רקע הצלחה)
  - אדום-כשל: `#A32D2D` (כשל), `#FCEBEB` (רקע כשל)
  - כתום-אזהרה: `#BA7517` (אזהרה), `#FAEEDA` (רקע אזהרה)
- דשבורד כפול-תפקיד: נתונים + גישה ישירה לפעולה באותו מסך
- breadcrumbs + שם דף בולט בכל מסך
- כל קלט נשמר ב-localStorage וניתן לעריכה בכל עת
- הסבר קצר לכל פעולה — המשתמש לא צריך לנחש

## סטטוסי ציות

```typescript
type ComplianceStatus = 'pass' | 'warn' | 'fail' | 'pending'
```

- `pass` — עומד בדרישה
- `warn` — חורג עד 20% מהדרישה, ניתן לתיקון
- `fail` — חריגה מהותית, חוסם היתר
- `pending` — ממתין לנתוני קלט

## כללי פיתוח

1. כל פרמטר בדיקה = רכיב `<ComplianceRow>` עצמאי
2. לא לשבור RTL — לבדוק כל רכיב חדש בעברית
3. כל חישוב — ב-`utils/calculations.ts`, לא בתוך רכיב
4. localStorage key: `ageplan_project_v1`
5. TypeScript strict mode — אין `any`
