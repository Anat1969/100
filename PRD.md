# PRD — AgePlan v1.0

## בעיה שפותרים

אדריכלים ומתכננים עירוניים מחויבים לתדריך משרד הבינוי (2024) שמגדיר 40+ פרמטרים כמותיים לתכנון ידידותי גיל. כיום הם פזורים בין תקני נגישות, הנחיות תמ"א ושיקול דעת אישי — אין כלי שמאחד ומאמת הכל בסביבת עבודה אחת. תוצאה: טעויות ציות שעולות בשלב ההיתר.

## משתמש יעד

אדריכל/ית שמכינ/ה תב"ע, תכנית מפורטת, או תכנית שכונה. זקוק/ה לדוח ציות מובנה שניתן לצרף להיתר בניה.

---

## מסכים — גרסה 1

### 1. לוח בקרה ראשי (Dashboard)
- סיכום ציות כולל באחוזים
- מספר פרמטרים: עוברים / אזהרות / כשלים / ממתינים
- כפתור "צור דוח PDF"
- ניווט לכל שכבת תכנון

### 2. בדיקת ציות — רצועת הליכה
- קלט: רוחב רצועה, סיווג רחוב (שכונתי/מסחרי)
- קלט: שיפוע, אחוז הצללה, מרחק בין ספסלים
- פלט: סטטוס לכל פרמטר (עובר / אזהרה / כשל)
- ויז'ואל: חתך רחוב סכמטי SVG

### 3. בדיקת ציות — מרחבי מפגש
- קלט: מרחק בין ספסלים, מרחק לחדר נוחיות
- קלט: מקומות ישיבה ביחס ליח"ד בבניין
- קלט: קיום מסלול מעגלי שכונתי
- פלט: סטטוס + ממצאים ספציפיים

### 4. בדיקת ציות — פארקים וגינות
- קלט: שטח פארק (דונם), פריסת ספסלים, חדרי נוחיות
- קלט: מתקני כושר, הצללה, תאורה
- פלט: סטטוס + רשימת חסרים

### 5. דוח ציות PDF
- שם פרויקט + תאריך
- טבלת ציות מלאה לפי שכבות
- ממצאים וחסרים מפורטים
- הפניה לסעיף התדריך הרלוונטי לכל כשל

## מה לא בגרסה 1 (v2+)

- קריאת קבצי DWG/IFC
- ניתוח הצללות אוטומטי (GIS)
- מולטי-יוזר / שיתוף פרויקטים
- API חיצוני לנתוני גבהים
- השוואה בין גרסאות תכנית

---

## הגדרת הצלחה

1. אדריכל יכול להזין פרמטרים ולקבל דוח ציות תוך 5 דקות
2. הדוח ניתן לצירוף ישיר להגשת היתר
3. כל כשל מפנה לסעיף ספציפי בתדריך

---

## זרימת משתמש ראשית

```
כניסה → יצירת פרויקט → קלט פרמטרים לפי שכבה
→ צפייה בסטטוס ציות → תיקון ממצאים → ייצוא PDF
```
